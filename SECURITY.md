# Security Policy

## Reporting a Vulnerability
Please do not report security vulnerabilities through public GitHub issues.

Open a private GitHub Security Advisory or email the maintainer directly.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix if known

We will acknowledge receipt within 48 hours and aim to release a fix within 14 days for critical issues.

## Scope
- The Cortex desktop application
- The Cortex module registry
- The brain database schema and encryption

## Out of Scope
- Third-party LLM providers (Claude, GPT, etc.)
- Third-party sync services (Dropbox, iCloud, etc.)
