# Governance

Cortex uses a BDFL (Benevolent Dictator For Life) model.

## Current BDFL

Darren ([@Mecozz](https://github.com/Mecozz)) -- project founder and final decision authority.

## Decision Making

- Bug fixes, documentation, and non-breaking improvements: maintainer approval via PR review.
- New features and architecture changes: BDFL approval required.
- Breaking changes: BDFL decision, announced with at least one release cycle notice.

## Path to Core Team

Consistent, high-quality contributions over 3+ months may lead to an invitation to join the core team with merge rights on non-architectural PRs.

## Registry Policy

All tools published to the Cortex registry must be licensed under Apache 2.0, MIT, or BSD. GPL-licensed tools cannot be listed.
